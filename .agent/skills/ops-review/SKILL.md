# 🔍 코드 리뷰어 (Code Reviewer)

이 프로젝트의 코드를 리뷰할 때 아래 체크리스트를 따르세요.

## 리뷰 체크리스트

### 1. 에러 핸들링

- Gemini API 호출: 429/503 시 키 로테이션이 동작하는가?
- FAL AI 호출: 응답 실패 시 사용자에게 명확한 에러 메시지를 보여주는가?
- Notion API 호출: 413 payload 초과 방지가 되어 있는가?
- catch 블록에서 에러를 삼키지 않고 로깅하는가?

### 2. 보안

- .env.local의 API 키가 클라이언트에 노출되지 않는가? (서버 사이드만 사용)
- base64 이미지가 노션에 직접 전송되지 않는가?
- 사용자 입력이 프롬프트 인젝션에 취약하지 않은가?

### 3. 성능

- 이미지 압축이 적용되어 있는가? (compressImage 함수)
- 불필요한 리렌더링이 없는가?
- API 호출 중 중복 요청 방지가 되어 있는가?

### 4. 코드 품질

- DRY 위반이 없는가? (같은 로직이 여러 곳에 있으면 안 됨)
- TypeScript 타입이 명확한가? (any 남용 금지)
- 컴포넌트가 너무 크지 않은가? (500줄 이상이면 분리 고려)

### 5. 프롬프트 품질 (Gemini)

- JSON 응답 형식이 명확히 지정되어 있는가?
- 프롬프트에 역할(persona)이 설정되어 있는가?
- 환각 방지 지침이 포함되어 있는가?

## 리뷰 결과 형식

```
STATUS: PASS / ISSUES_FOUND / CRITICAL
FINDINGS:
1. [파일:라인] 설명 — 심각도(HIGH/MED/LOW)
2. ...
RECOMMENDATION: 수정 방향
```
