# 💡 기획 리뷰어 (Plan Review / CEO Mode)

새 기능을 추가하기 전에 아래 질문에 답하세요. gstack의 CEO Review 컨셉을 이 프로젝트에 맞게 적용한 것입니다.

## 전제 검증 (Premise Challenge)

1. 이 기능이 해결하는 실제 문제는 무엇인가?
2. 이 기능 없이도 사용자가 목표를 달성할 수 있는가?
3. 아무것도 안 하면 어떤 일이 벌어지는가? (진짜 고통인가, 가상의 문제인가?)

## 타겟 사용자 검증

- 이 기능의 1차 사용자는 누구인가? (인스타 셀러 / 소규모 브랜드 / 마케팅 에이전시)
- 그 사용자가 이 기능에 돈을 낼 의향이 있는가?
- 경쟁 도구 대비 차별점은 무엇인가?

## 기존 코드 활용

- 이미 존재하는 코드로 부분적으로 해결 가능한가?
- 새로 만드는 것보다 기존 코드를 확장하는 게 나은가?

## 수익화 연결

- 이 기능이 Free/Pro/Agency 중 어느 플랜에 해당하는가?
- 이 기능이 유료 전환을 유도하는가?
- 이 기능이 기존 사용자의 리텐션을 높이는가?

## 구현 대안 (최소 2개)

```
접근법 A: [이름]
  요약: [1~2문장]
  노력: S/M/L
  리스크: Low/Med/High
  장점: [2~3개]
  단점: [2~3개]

접근법 B: [이름]
  요약: [1~2문장]
  노력: S/M/L
  리스크: Low/Med/High
  장점: [2~3개]
  단점: [2~3개]
```

## 12개월 후 비전

```
현재 상태 → 이 기능 추가 후 → 12개월 후 이상적 상태
```

## 결정

- PROCEED: 진행 (이유)
- DEFER: 나중에 (이유)
- KILL: 안 함 (이유)

## 이 프로젝트의 핵심 판단 기준

1. 인스타 셀러/브랜드가 실제로 쓸 기능인가?
2. Gemini API 비용 대비 가치가 있는가? (Free Tier 한도 고려)
3. mobileeditingclub 수준의 결과물을 낼 수 있는가?
4. 노션 연동이 자연스러운가?
5. 30분 안에 구현 가능한가? (AI 코딩 시대의 기준)
